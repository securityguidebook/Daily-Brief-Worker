# Daily Brief — Deployment Guide

## Repo structure

```
daily-brief-worker/       ← this repo, deploy to Cloudflare
  src/index.js            ← the Worker code
  wrangler.toml           ← Cloudflare config (cron, KV binding)
  package.json

daily-briefing-agent.html ← your GitHub Pages frontend
```

---

## Step 1 — Install Wrangler

```bash
npm install -g wrangler
wrangler login
```

---

## Step 2 — Create the KV namespace

KV is Cloudflare's key-value store. It caches your latest brief so
Daily Digivolve can fetch it on demand without regenerating.

```bash
npx wrangler kv:namespace create BRIEF_CACHE
```

Copy the `id` it prints and paste it into `wrangler.toml`:

```toml
[[kv_namespaces]]
binding = "BRIEF_CACHE"
id = "PASTE_YOUR_ID_HERE"
```

---

## Step 3 — Set your secrets

Never commit these to Git. Wrangler stores them encrypted in Cloudflare.

```bash
npx wrangler secret put ANTHROPIC_API_KEY
# paste your Anthropic API key when prompted

npx wrangler secret put ALLOWED_ORIGIN
# paste your GitHub Pages URL, e.g. https://securityguidebook.github.io
```

---

## Step 4 — Deploy the Worker

```bash
npm install
npx wrangler deploy
```

Cloudflare will give you a Worker URL like:
`https://daily-brief-worker.YOUR_NAME.workers.dev`

---

## Step 5 — Update the HTML app

In `daily-briefing-agent.html`, find this line and replace it:

```js
const WORKER_URL = 'https://daily-brief-worker.YOUR_NAME.workers.dev';
```

Push the updated HTML to your GitHub Pages repo. Done.

---

## Step 6 — Verify cron times

The Worker fires at:
- `0 18 * * *` UTC = **6:00 AM NZT** (UTC+12)
- `0 5 * * *`  UTC = **5:00 PM NZT**

You can verify in the Cloudflare dashboard under Workers → your Worker → Triggers.

To test the cron manually without waiting:
```bash
npx wrangler dev --test-scheduled
# then in another terminal:
curl "http://localhost:8787/__scheduled?cron=0+18+*+*+*"
```

## Daily Digivolve API integration

The Worker exposes two endpoints for your webapp:

### GET /brief
Returns the latest cached brief (generated by cron or manually).
No parameters needed. Call this when your avatar wakes up.

```js
const res = await fetch('https://daily-brief-worker.YOUR_NAME.workers.dev/brief');
const brief = await res.json();

// brief.summary  → short string for avatar greeting
// brief.markdown → full formatted brief
// brief.date     → human-readable date
// brief.generatedAt → ISO timestamp
```

**Avatar greeting example:**
```js
avatar.say(brief.summary);
// → "Today's top stories: CVE-2025-XXXX hits Cisco · EU AI Act enforcement begins · NZ election update"
```

### POST /generate
Triggers a fresh generation on demand (e.g. user taps "refresh" in Digivolve).

```js
const res = await fetch('https://daily-brief-worker.YOUR_NAME.workers.dev/generate', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    topics: ['Cybersecurity', 'AI & Machine Learning'],
    depth: 3,
    tldrGlobal: true,
    tldrEu: true,
  })
});
const brief = await res.json();
```

### GET /health
Quick check that the Worker is up.

## Cost estimate

| Service           | Free tier          | Likely usage        |
|-------------------|--------------------|---------------------|
| Cloudflare Worker | 100k req/day       | ~60/month (cron×2 + manual) |
| Cloudflare KV     | 100k reads/day     | Negligible          |

You'll comfortably stay within Cloudflare's free tier.

---

## Security notes

- API key is stored as a Cloudflare secret — never in code or Git
- CORS is locked to your `ALLOWED_ORIGIN` — other origins are blocked
- No user data is stored — KV only holds the generated brief text
- Consider adding a simple bearer token to `/generate` before exposing
  it to Daily Digivolve in production (prevents anyone triggering generations)
